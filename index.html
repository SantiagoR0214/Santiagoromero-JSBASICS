JavaScript Asynchronous Programming

Async programming allows functions to run without waiting for other operations to finish. This makes your code more efficient and responsive.

// Example: Basic async function
async function fetchData() {
    let data = await fetch('https://api.example.com/data');
    return await data.json();
}
fetchData().then(console.log);

Working with Callbacks

Callbacks are an older, but effective way to handle asynchronous tasks. By passing a function as a parameter, you can execute it once the main function has finished.

// Example: Callback function
function fetchData(callback) {
    setTimeout(() => {
        console.log("Data retrieved");
        callback();
    }, 1000);
}

fetchData(() => {
    console.log("Callback executed");
});

Promises in JavaScript

Promises provide a cleaner way to manage async operations. They can be resolved or rejected, allowing you to handle success and failure cases.

// Example: Promise
let myPromise = new Promise((resolve, reject) => {
    setTimeout(() => resolve("Task successful"), 1500);
});

myPromise.then(result => console.log(result));

Async and Await

The `async` and `await` keywords offer an elegant solution for writing asynchronous code. Async functions return promises, and await pauses execution until a promise resolves.

// Example: Async/Await
async function getData() {
    let response = await fetch('https://api.example.com/resource');
    let data = await response.json();
    console.log(data);
}
getData();

Understanding the Event Loop

JavaScript's event loop allows it to handle asynchronous tasks efficiently, ensuring non-blocking execution even with a single-threaded environment.

// Example: Event loop in action
console.log('Start');

setTimeout(() => console.log('Async task completed'), 2000);

console.log('End');
